Which AI service should power the content summarization?

Used to generate plain-English summaries and "why it matters" notes during ingestion. Runs in the background — not user-facing.

A

OpenAI (GPT-4o mini)

Fast, cheap, widely used. GPT-4o mini is well-suited for structured summarization tasks. Easy to integrate. ~$0.15/1M input tokens.

Best for: Cost-sensitive, high-volume summarization pipelines.

B

Anthropic Claude (Haiku)

Strong at following structured prompts and producing consistent output. Claude Haiku is fast and affordable. ~$0.08/1M input tokens.

Best for: Reliable structured output with good instruction-following.

C

Decide later — stub it out for MVP

Build the pipeline with a placeholder summarizer that returns the raw excerpt. Wire in a real AI service once the pipeline is stable and you've validated the ingestion flow.

Best for: Getting the pipeline running fast without committing to an API key yet.
